Консоль зарускать в папке samodelka-mod!



Как загрузить последнюю версию мода
?
Вводим в консоль: $ git pull origin master



Как отправить свои нароботки?

Вводим в консоль: $ git add . (добавляем все изменения в коммит)

$ git commit -m "какой-нибудь комментарий" (сохраняем изменения)

$ git push origin master (отправляем изменения)



Когда производить загрузку и отправку?

Загрузку делать на всякий случай каждый раз перед началом работы.

Отправку совершать после каждого сеанса работы. 



Что делать, если нужно создать новый репозиторий?

Вводим в консоль: $ git init (создание репозитория)

$ git remote add origin https://gitlab.com/sanchello13pv/samodelka-mod.git (установление доступа к удалённому репозиторию, куда сливаются наработки)

$ git pull origin master (заргузка файлов из удалённого репозитория)



Как перемещаться между коммитами (различными версиями)?

Вводим в консоль: $ git log --oneline (просмотр всех коммитов, содержание характеризуется оставленными комментариями)

$ git checkout "название комиита" (перемещение на нужный коммит)

$ git checkout master (перемещение на последний коммит (если указатель "master" не перемещали))



Если при загрузке появляется сообщение, содержащее слова merge conflict и в скобках 
появилась голубая надпись merge, то необходимо открыть файл, в котором появился данный
конфликт и переписать его в зависимости от твоих потребностей.

В файле должны присутствовать части кода, окаймлённые по следующему образу и подобию:



<<< HEAD

*какая-то часть программного кода*

========
*
какая-то часть программного кода*
>>> *название коммита из различных сочетаний букв и цифр*



Если необходимо добавить свои нововведения с заменой, то часть кода от надписи "HEAD"

до полосок равно включительно нужно удалить. Если же необходимо только добавить, то

копируем часть кода от заголовка "HEAD" до "названия коммита" и вставляем его сразу

после этого коммита. В первой части (от куда копировали) удаляем все фрагменты с 

заголовком "HEAD", а во второй части (что вставили) удаляем все фрагменты с

заголовком "название коммита". Сохраняем файл.

Чтобы такой хернёй не маяться, лучше всего работать в отдельных режимах разработки:

один - редактирует карту (F2), а второй - мутит интерактив (F3), тогда конфликтов 
не возникнет.
